# 新しい資料の作り方（オーサリングガイド）

このリポジトリの教材は **Markdown だけ書けば作れます**。HTML・QRコード・もくじ（index）は
`build.py` が自動生成するので、`docs/` の中を直接さわる必要はありません。

**1つの章 = 1つのフォルダ**です。フォルダの中に本文（index.md）と画像（img/）を入れます。

```
content/                    ← ここだけ編集する
  _template/index.md        ← 新しい章のひな形
  b1_intro/                 ← 章フォルダ（フォルダ名 = <id>_<slug>）
    index.md                ←   本文（Markdown）
    img/                    ←   画面キャプチャ
templates/                  ← HTMLのひな形（基本さわらない）
build.py                    ← 変換スクリプト
docs/                       ← 自動生成される公開ファイル（さわらない）
  b1_intro/{index.html, qr.svg, img/}   ← content と同じ形で出力される
  assets/{deck.css, deck.js, hakase.png, qr.svg}
  index.html
```

章フォルダ名の頭は **b=基礎編 / e=拡張編 / mq=ロボットカー編** ＋ 連番（例：`b6_temp`, `e2_servo`, `mq4_remote`）。

---

## 手順（4ステップ）

1. **ひな形フォルダをコピー**して新しい章を作る
   ```bash
   cp -r content/_template content/e2_温度センサー
   ```
2. `content/e2_温度センサー/index.md` を**Markdownで書く**（下の「書き方」参照）
3. 画面キャプチャを `content/e2_温度センサー/img/` に入れる（必要なら矢印などを付ける → 「マーキング」参照）
4. **ビルドして確認 → 公開**
   ```bash
   python build.py                 # docs/ を再生成（HTML・QR・もくじ）
   # docs/e2_温度センサー/index.html をブラウザで開いて確認
   git add -A && git commit -m "Add chapter e2" && git push
   ```
   push すると数十秒で GitHub Pages に反映されます。

> はじめて使うときだけ: `pip install segno pillow`（QR生成と画像縮小に使います）

## 書き方

### フロントマター（ファイル先頭の設定）

| 項目 | 説明 | 例 |
|---|---|---|
| `id` | 章の番号。ファイル名・QR・画像フォルダに使う | `06` / `mq4` |
| `slug` | URLになる英語名 | `thermo` |
| `section` | もくじのどのセクションに入れるか | `基礎編` / `拡張編` / `ロボットカー編` |
| `emoji` | もくじカードの絵文字 | `🌡️` |
| `num` | カードの番号バッジ | `6` / `④` |
| `color` | カードの枠色 | 基礎/拡張は `c1`〜`c5`、ロボットは `r1`〜`r3` |
| `nav_title` | 画面上部・タブに出るタイトル | `⑥ 温度センサーを使おう` |
| `card_title` | もくじカードの見出し | `温度センサーを使おう` |
| `desc` | もくじカードの説明（1〜2文） | `気温をはかって…` |
| `level` | （ロボットカー編だけ）難易度ラベル | `入門編` |

### スライドの記法（本文）

| 書き方 | 意味 |
|---|---|
| `---`（行頭・単独） | スライドの区切り |
| `{cover}` | 表紙レイアウト（中央寄せ）。QRは自動で右下に付きます |
| `# 見出し` | スライドのタイトル |
| `:::` | ここで左右2カラムに分ける（左=説明、右=画像 が基本） |
| `1. ` / `- ` | 番号つき手順／箇条書き |
| `**太字**` `` `コード` `` | 強調・コード表示 |
| `![キャプション](img/xxx.png)` | 画像（連続で書くと横並び） |
| `> ふつうの文` | 博士のヒント吹き出し（緑） |
| `> ⚠ 文` | 注意の吹き出し（オレンジ） |
| `> 💪 文` | チャレンジの吹き出し（青） |

**定番の並び**は「表紙 →『なにをするの？』→ 手順①②… 」です。`_template/index.md` がその形になっています。

## 画面キャプチャの渡し方

- その章のフォルダの **`img/`** に入れる（例：`content/e2_温度センサー/img/step1.png`）
- 本文からは **`img/ファイル名`** で参照（`![説明](img/step1.png)`）
- ビルド時に `docs/<章>/img/` へコピーされ、大きすぎる画像は自動で縮小されます
- 名前は順番がわかるように `01_open.png` `02_block.png` … と付けると迷いません
- 形式は画面キャプチャなら **PNG** がおすすめ

## マーキング（矢印・囲み・「ここをクリック」）

画像に矢印や囲みを付けたいときは、**Greenshot**（Windows・無料）が手軽でおすすめです。

### Greenshot のセットアップ
1. https://getgreenshot.org/ からダウンロードしてインストール
2. `PrintScreen` キーで範囲を選んでキャプチャ → そのまま注釈エディタが開く
3. ツールバーで **矢印・四角・ハイライト・番号・文字** を追加
4. 「名前を付けて保存」で `content/<id>/img/` に PNG 保存

> **ShareX**（https://getshar.com/ ）も同様に無料で高機能です。お好みで。

### マーキングのコツ（子ども向け）
- 押す場所は **赤い四角で囲む** ＋ 短い言葉（「①ここをクリック」）
- 1枚に矢印を入れすぎない（多くても2〜3個）
- そもそも本文の「**番号つき手順**」＋スクショで大半は伝わるので、マーキングは"ここぞ"だけに

## もくじのセクションについて

`section` に書いた名前でカードが自動的に振り分けられます。

- `基礎編` … micro:bit だけ（外部デバイスなし）
- `拡張編` … 外部デバイスをつなぐ
- `ロボットカー編` … micro:Maqueen（`level` も付ける）

順番は `id` の昇順です。新しいセクションを増やしたいときは `build.py` の `SECTIONS` に追記してください。
